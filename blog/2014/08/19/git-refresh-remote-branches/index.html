Git でリモートリポジトリの更新が反映されないとき

GitHub のWebサイトでブランチを削除したあと、クライアント（SourceTreeとか）のリモートブランチの表示に、削除したはずのブランチが残っていて、気持ち悪いなあ、と思っていた。

git のコマンド一発だった。

git remote update -p

これでクライアント側のリモート情報がリフレッシュされる。
